## Quick Start for GitHub Contributors

### 1. Small Fixes (typos, broken links, clarifications)
- Open an Issue describing the fix needed
- Submit a Pull Request once approved

### 2. Substantial Changes (new practices, significant rewrites)
- Start a Discussion first to gather input
- Collaborate on approach before submitting PR

### 3. Pull Request Process
1. Fork the repository
2. Create a descriptive branch name
3. Make your changes following existing style
4. Commit with clear messages
5. Submit PR referencing any related Issues/Discussions
6. Engage constructively in review

## Contribution Areas

- **Practice refinements** from real implementation
- **Translations** into other languages
- **Case studies** from pilot organizations
- **Templates and resources** for facilitation
- **Clarifications** of Islamic or Agile concepts
